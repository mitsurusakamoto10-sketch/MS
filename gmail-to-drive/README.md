# Gmail → Markdown → Google ドライブ 変換

指定した **Gmail ラベル（フォルダ）** 内のメール本文を **Markdown(.md)** に変換し、
指定した **Google ドライブのフォルダ** に保存する Google Apps Script です。
**毎日決まった時刻に自動実行**でき、**処理済みのメールは重複保存しません**。

## ファイル構成

```
gmail-to-drive/
├── Code.gs           … 変換・保存・スケジュール登録の本体
├── appsscript.json   … GAS マニフェスト（権限スコープ・タイムゾーン）
└── README.md         … この手順書
```

## 仕組み・重複防止

メール 1 通ごとに以下を行います。

1. 対象ラベルが付き、かつ処理済みラベルが付いていないメールだけを検索
2. 本文（プレーンテキスト）を Markdown 化し、ドライブに `.md` として保存
3. 処理済みラベルを付与し、メッセージ ID を記録

重複は **3 段構え**で防ぎます。

- Gmail 検索で「処理済みラベル付き」を除外
- Script Properties に保存したメッセージ ID と照合
- ドライブ上の同名ファイル有無をチェック

ファイル名は `yyyy-MM-dd_件名_メッセージID.md` 形式で、メッセージ ID により必ず一意になります。

## セットアップ手順

### 1. プロジェクト作成

1. <https://script.google.com> を開き「新しいプロジェクト」を作成
2. エディタの `コード.gs` を本リポジトリの `Code.gs` の内容で置き換える
3. （任意）「プロジェクトの設定」→「`appsscript.json` を表示」を有効化し、
   `appsscript.json` の内容を反映（タイムゾーン・権限スコープ）

> `clasp` を使う場合は、このフォルダごと `clasp push` すれば 2 ファイルを反映できます。

### 2. 設定値（CONFIG）の編集

`Code.gs` 冒頭の `CONFIG` を自分の環境に合わせます。

| 項目 | 説明 |
| --- | --- |
| `GMAIL_LABEL` | 変換対象の Gmail ラベル名（フォルダ）。ネストは `親/子` |
| `DRIVE_FOLDER_ID` | 保存先フォルダ ID。URL の `/folders/` 以降の文字列 |
| `PROCESSED_LABEL` | 処理済みに付けるラベル名（無ければ自動作成） |
| `DAILY_HOUR` | 毎日の実行時刻（0〜23、タイムゾーン基準） |
| `MAX_THREADS` | 1 回で処理する最大スレッド数 |

### 3. 権限承認（初回のみ）

エディタで関数 `convertGmailToDrive` を選び「実行」。
初回は Gmail / ドライブへのアクセス許可を求められるので承認します。
対象ラベルにメールがあれば、ドライブに `.md` が作成されます。

### 4. 毎日の自動実行を登録

関数 `setupDailyTrigger` を一度実行すると、`CONFIG.DAILY_HOUR` 時に
毎日 `convertGmailToDrive` が走る時刻トリガーが登録されます。
（再実行しても既存トリガーを作り直すので重複しません）

自動実行を止めたいときは `removeDailyTrigger` を実行します。

## 主な関数

| 関数 | 役割 |
| --- | --- |
| `convertGmailToDrive` | メイン処理（変換・保存）。トリガーからも呼ばれる |
| `setupDailyTrigger` | 毎日実行のトリガーを登録 |
| `removeDailyTrigger` | 毎日実行のトリガーを削除 |

## 注意点

- 本文は**プレーンテキスト**を Markdown 化します（HTML 装飾・添付は対象外）。
- 1 通＝1 ファイルです。同じスレッドの複数メールは個別に保存されます。
- `MAX_THREADS` を超える分は次回実行に持ち越されます（取りこぼしなし）。
