# ホテル供給状況レポート自動生成

メトロエンジンの生データ（既存施設 / 新規開業予定施設 / 閉業施設の3CSV）から、
社内マニュアルの作業フロー（グレード付け→閉館反映→年度別積み上げ集計→グラフ化）を
再現したExcelレポートを生成する。総合職の担当者が新規都市のデータを自分で作成・
メンテできる簡易フローを目的とする。

## 使うもの

- 本体スクリプト: `tools/hotel_supply/make_supply_report.py`（依存: `openpyxl`、LLM調査時のみ `anthropic`）
- 詳細手順書（VS Code含む）: `tools/hotel_supply/VSCODE_MANUAL.md` … 迷ったら必ずこれを読む
- 受け渡しデータ（リポジトリ非管理・スクリプトと同フォルダに置く）:
  `grade_rules.csv`（ブランド→グレード表）, `<都市>_grade_overrides.csv`（施設別グレード）,
  `grade_research_cache.csv`（LLM調査キャッシュ）

## 手順

1. **入力CSVを確認する**。3ファイルが揃っているか、列名が想定形式か（部分一致で吸収）:
   - 既存: `施設名, 住所, カテゴリー, 部屋数, 開業, プライス・インデックス`
   - 新規: `ホテルID, 施設名, 住所, (推定)部屋数, 竣工・開業予定日`
   - 閉業: `閉業施設名, 住所, 部屋数, 閉業日, カテゴリー`（部屋数に「集計中」あり）

2. **実行する**:
   ```bash
   cd tools/hotel_supply
   python3 make_supply_report.py --city <都市名> \
     --existing 既存.csv --new 新規.csv --closed 閉業.csv \
     [--grade-overrides <都市>_grade_overrides.csv] [--asof YYYY-MM]
   # → YYYYMMDD_供給レポート_<都市名>.xlsx（冒頭は作成日の西暦8桁）
   ```
   - `--no-llm`: LLM調査せずキャッシュのみ使用（APIキー不要）
   - `--list-unresolved`: ルールで決まらない施設だけ一覧表示（調査前の確認に）
   - `--refresh-research`: キャッシュを無視して再調査

3. **グレード判定はこの優先順**（1つ確定したら以降に進まない）:
   1. 施設別グレード表（オーバーライド）に同名があるか
   2. カテゴリーが宿泊主体型（ビジネス/デラックス/シティ/リゾートホテル・旅館）以外 → 不明・その他
   3. ブランド→グレード表に一致 → そのグレード
   4. 20室未満（既存のみ） → D
   5. **LLM調査（Web検索）で主力客室=ボリュームゾーンの面積を調べ判定**:
      A=40㎡以上 / B=30㎡以上 / C=20㎡以上 / D=20㎡未満（面積優先、単価は補助）
   6. それでも不明 → 不明・その他＋要確認（黄色行）
   - **再現性**: LLM結果は `grade_research_cache.csv` に面積・根拠・出典つきで保存され、
     次回以降は必ず再利用（毎回同じ回答）。判定を直すときはこのCSVを編集する。

4. **集計ロジック**（年度=4月〜翌3月）: 各年度Yの供給室数 =
   「開業年度≤Y かつ（未閉業 or 閉業年度>Y）」の既存部屋数合計 ＋「開業年度≤Y」の新規供給累積。
   開業日不明の既存は2019年度以前開業とみなす。**閉業年度以降は自動的に供給から除外される**。

5. **検証する**（VS Codeでもここは必ず）:
   - 出力xlsxのzip整合性、7シート構成（供給数推移/既存/新規/閉業/参考3種/フロー）を確認
   - 合計室数が妥当な規模か、閉業が減算されているか（`--list-unresolved`や別計算で突合）
   - 生成物は1つのExcelにまとまる（参考CSVは参考タブ、判定手順は「グレード判定フロー」タブ）

## 出力Excelの構成

`{都市} 供給数推移`（集計表＋積み上げ棒＋円グラフ）/ `既存リスト` / `新規供給リスト` /
`閉業リスト`（各グレード列はプルダウン）/ `参考_ブランドグレード表` / `参考_施設別グレード表` /
`参考_LLM調査キャッシュ` / `グレード判定フロー`（自然言語の説明）。
集計表はリストシートへのSUMIFS数式なので、リスト上でグレード・年度を手修正すると表・グラフが自動再計算される。

## 注意

- メトロエンジンのログインID/パスワード等の資格情報は、生成物・コミットに一切含めない。
- 入力CSV・グレード表・出力Excelは社内情報。リポジトリにコミットしない（`.gitignore`済み）。
- 分析はメトロエンジンのデータに完全準拠（衛生行政報告例との突合は行わない）。
- グレード列（既存/新規/閉業）はExcel上のプルダウンで手修正可能。修正は集計・グラフに自動反映。

詳細（インストール、トラブル対処、列名対応、他都市展開、判定基準の背景など）は
`tools/hotel_supply/VSCODE_MANUAL.md` を参照。
