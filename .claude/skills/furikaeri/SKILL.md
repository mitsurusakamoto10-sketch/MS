# 振り返りワークフロー

PLAUD NOTE の録音（文字起こし）を、対話で補足しながら議事録mdに整え、Google Drive「mdファイル」フォルダへ保存する。

## 前提

- PLAUD NOTE MCP（`mcp__Plaud_Note__*`）と Google Drive MCP（`mcp__Google_Drive__*`）が接続されていること。未接続なら ToolSearch で探し、なければユーザーにコネクタの接続を依頼する。
- 保存先: Drive の「mdファイル」フォルダ（初回に `search_files` で `name contains 'mdファイル' and mimeType = 'application/vnd.google-apps.folder'` を検索してIDを特定。複数候補があればユーザーに確認）
- 話し方フィードバックの保存先: 「mdファイル/話し方フィードバック/」サブフォルダ（なければ作成を試み、作成できなければユーザーに手動作成を依頼）

## 手順

### 1. 未処理録音の洗い出し

1. `mcp__Plaud_Note__list_files` で録音一覧を取得（初回運用は直近1ヶ月分。2回目以降は前回処理分以降、目安として直近2週間を確認）
2. 各録音について、Drive の `search_files` で fullText に `plaud_file_id` の値が含まれる md を検索
3. ヒットしないものが「未処理」。未処理一覧（日時・タイトル・長さ）をユーザーに提示する

### 2. 1件ずつ処理（未処理分をループ）

1. `mcp__Plaud_Note__get_transcript` で文字起こし、`mcp__Plaud_Note__get_note` でAI要約を取得
2. 内容の要点を短く提示した上で、**チャットで補足情報を質問する**:
   - いつの、誰との、どんな場（会議/電話/打合せ/雑談）の会話か
   - 話者ラベルのうち、どれがユーザー本人か（話し方フィードバックに必須）
   - 議事録に残すべき文脈・背景（案件名、前提など）
   - 内容から推測できる部分は推測を提示して確認だけ求める（ユーザーの手間を最小化）
3. 補足が揃ったら議事録mdを生成（テンプレートは下記）
4. `mcp__Google_Drive__create_file` で「mdファイル」フォルダへ保存
   - `disableConversionToGoogleType: true`、mimeType は text/markdown
   - ファイル名: `YYYY-MM-DD_相手_タイトル.md`
5. 話し方フィードバックmdを生成し「mdファイル/話し方フィードバック/」へ保存
   - ファイル名: `YYYY-MM-DD_タイトル.md`
6. 保存後、`get_file_metadata`等で保存結果を確認し、ユーザーに報告して次の1件へ

件数が多い場合は数件ずつ区切り、続けるかユーザーに確認する。

### 3. 完了報告

処理した件数、保存したファイル名一覧、スキップしたもの（処理済み・文字起こしなし等）を報告する。

## 議事録mdテンプレート

4つの目的（知識蓄積・話し方FB・仕事の記録・議事録作成）を満たすこと。目的が変わったら CLAUDE.md とこのテンプレートを更新する。

```markdown
---
plaud_file_id: <PLAUDのファイルID>
日時: YYYY-MM-DD HH:MM
種別: 会議 / 電話 / 打合せ / 雑談
相手・参加者: <補足確認した内容>
場所・文脈: <補足確認した内容>
tags: [<案件名や分野>]
---
# <タイトル>

## 概要
（3行以内）

## 議事内容（要点）

## 決定事項

## タスク
### 自分が受けた指示・依頼
| 依頼者 | 内容 | 期限 |
### 自分が依頼した仕事
| 担当 | 内容 | 期限 |

## 知識・学び

## 未確定・持ち越し事項
```

## 話し方フィードバックmdテンプレート

ユーザー本人の発言のみを分析対象とする。励ましつつ率直に。

```markdown
---
対象議事録: <議事録mdのファイル名>
日時: YYYY-MM-DD
---
# 話し方フィードバック: <タイトル>

## 良かった点
## 気になった癖（口癖・フィラー・言い回し）
（実際の発言を短く引用して指摘する）
## 話の構成・伝わりやすさ
## 言葉遣い・ビジネス表現
## 次回の改善アクション（1〜3個）
```

## 絶対ルール

- 文字起こし・議事録の内容をこのリポジトリにコミットしない
- 保存前に生成したmdの内容をユーザーに提示し、修正希望がないか確認する
- 同じ `plaud_file_id` のmdを重複作成しない
