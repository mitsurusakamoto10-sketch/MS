# このリポジトリでのClaudeの動作ルール

このリポジトリは2つの役割を持つ。

1. **マイポータル（静的サイト）**: index.html / style.css / script.js / functions/ で構成される Cloudflare Pages サイト
2. **PLAUD NOTE 振り返りワークフローの拠点**: PLAUD NOTE と Google Drive の MCP 連携を使った議事録化・知識蓄積の手順を管理する

## トリガーワード

### 「振り返り」

ユーザーが「振り返り」とチャットしたら、`.claude/skills/furikaeri/SKILL.md` のワークフロー（経路A）を実行する。
概要: PLAUD NOTE の未処理録音を探す → 文字起こしを取得 → **構造化ヒアリング**（必須8項目。全文から推定できる情報は確認に留め、欠落・要確認だけを選択式で質問）で補足情報を確認 → 整理版md（＋必要に応じ原文md・知識ノート）と話し方フィードバックmdを生成 → Google Drive「mdファイル」フォルダへ保存。

### 任意情報のmd化（貼付けの文字起こし/メモ）

ユーザーが任意の情報（文字起こし・メモ）を「md化して」「知識として残して」等と依頼したら、同スキルのワークフロー（経路B）を実行する。全文を読み → 構造化ヒアリング（8項目）→ 整理版md（＋選択に応じ原文md・知識ノート）を生成・保存する。話者属性のないメモは話し方FBを原則スキップ。

### 過去の会議・電話に関する質問（ナレッジQ&A）

ユーザーが過去の会議・電話・打合せの内容について質問したら:
1. Google Drive MCP の `search_files`（fullText検索、対象は「mdファイル」フォルダ配下）で該当する議事録mdを検索
2. `download_file_content` で内容を読み取り
3. 議事録に基づいて回答する（出典のファイル名・日付を添える）
4. 見つからない場合は PLAUD NOTE 側の文字起こしも検索してよい

## ワークフローの目的（md生成時は常にこれを満たすこと）

1. **知識蓄積**: 日々の会話の議事を知識として蓄積し、いつでも振り返れるようにする
2. **話し方フィードバック**: 文字起こしからユーザーの話し方の癖を分析し、より能力の高いビジネスマンになるための話し方・言葉遣いのフィードバックを行う
3. **仕事の記録**: 指示を受けた仕事・依頼した仕事を記録し、誤解が生まれないようにする
4. **議事録作成**: 定期的に必要となる会議の議事録を作成する
5. **知識資産化**: 再利用できる学び・ノウハウを「知識ノート」として蒸留し、担当交代でも失われない組織知にする

※ 目的は流動的に変更され得る。変更されたらこのファイルとスキルを更新すること。

## 保存先・命名規約

Google Drive「mdファイル」フォルダをルートとし、用途別に保存する。

- 整理版md（会議録/セミナーメモ/相談記録/備忘メモ）: 「mdファイル」直下、`YYYY-MM-DD_相手_タイトル.md`（相手が役割/不明なら省略可。日時不明時は `NoDate_タイトル.md`）
- 話し方フィードバック: 「mdファイル/話し方フィードバック/」配下、`YYYY-MM-DD_タイトル.md`
- 知識ノート: 「mdファイル/知識ノート/」配下、`YYYY-MM-DD_タイトル.md`
- 原文（文字起こし/メモ全文を別保存する場合）: 「mdファイル/00_Inbox/」配下、`YYYY-MM-DD_タイトル_原文.md`
- サブフォルダは使用時に存在確認し、無ければ作成する（作成不可なら手動作成を依頼）
- 保存時は必ず `disableConversionToGoogleType` を有効にし、純粋な .md（text/markdown）として保存する
- 保存種別はファイル名に入れず frontmatter で表現する

整理版mdのYAMLフロントマター（詳細はスキルのテンプレート参照）:

- `plaud_file_id`（PLAUD由来のみ。処理済み判定に使用）／任意情報は `source` を持たせる
- `保存種別`（会議録/セミナー・説明会メモ/相談記録/備忘メモ/知識ノート/その他）
- `日時`（不明可）、`場所`（オンライン/社内会議/外部セミナー/不明）、`相手・参加者`（実名/役割のみ/不明）
- `areas`（関連領域）、`tags`（案件名・分野）、`原文`（なし/本md内/00_Inbox参照）

## 処理済み判定

Drive MCP にファイル更新ツールはないため、インデックスファイルは使わない。
録音（PLAUD由来）が処理済みかどうかは、Drive の `search_files` で `plaud_file_id` の値を fullText 検索して判定する。ヒットしなければ未処理。補助としてファイル名規約（日付＋タイトル）でも照合する。
任意情報のmd化は単発のため重複判定は原則不要（必要なら日時＋タイトルで既存を確認する）。

## プライバシー・機密の絶対ルール

- 文字起こし・議事録・原文の**内容は絶対にこのリポジトリにコミットしない**（保存先はユーザーのGoogle Driveのみ）
- ポータル（公開サイト）に議事内容を表示する変更を、ユーザーの明示的な指示なしに行わない
