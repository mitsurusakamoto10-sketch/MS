# 市町村区ホテル供給状況レポート自動生成ツール

> **VS Codeなどローカル環境での詳細手順は [`VSCODE_MANUAL.md`](VSCODE_MANUAL.md) を参照**（インストール〜実行〜検証〜他都市展開まで）。
> Claude Code では `.claude/skills/hotel-supply-report/` の Skill として登録済みで、「供給レポート」「ホテルグレード」等の依頼で自動起動します。

メトロエンジンからダウンロードした3つのCSV（既存施設 / 新規開業予定施設 / 閉業施設）を
入力すると、**グレード別×年度別の供給室数推移（積み上げ棒グラフ）** と
**基準年度末のグレード別供給室数割合（円グラフ）** を含むExcelブックを自動生成します。

社内の「供給データ更新マニュアル」の作業フロー（グレード付け → 閉業反映 →
年度別積み上げ集計 → グラフ化）を1コマンドで再現するものです。

## 必要なもの

- Python 3.10以上
- `pip install openpyxl`（必須）
- `pip install anthropic` + 環境変数 `ANTHROPIC_API_KEY`（LLM調査を使う場合のみ）

## 入力CSV（メトロエンジンからDL / UTF-8）

| ファイル | 主な列 |
|---|---|
| 既存施設 | 施設名, 住所, カテゴリー, 部屋数, 開業, プライス・インデックス |
| 新規開業予定施設 | ホテルID, 施設名, 住所, (推定)部屋数, 竣工・開業予定日 |
| 閉業施設 | 閉業施設名, 住所, 部屋数, 閉業日, カテゴリー |

列名は部分一致で認識するため、多少の変更があっても動作します。

## データファイル（リポジトリには含めない・別途受け渡し）

スクリプトと同じフォルダに置いてください。**社内基準を含むためコミット禁止**
（`.gitignore`済み）。

| ファイル | 内容 |
|---|---|
| `grade_rules.csv` | ブランド→グレード対応表（社内グレードリストより。編集可） |
| `<都市>_grade_overrides.csv` | 施設名→グレードの個別対応表（任意。過去資料の判定を継承） |
| `grade_research_cache.csv` | LLM調査結果のキャッシュ（自動生成・編集可） |

## 使い方

```bash
python3 make_supply_report.py \
  --city 那覇市 \
  --existing 既存施設.csv \
  --new 新規開業予定施設.csv \
  --closed 閉業施設.csv \
  --grade-overrides naha_grade_overrides.csv
# → YYYYMMDD_供給レポート_那覇市.xlsx（冒頭は作成日の西暦8桁）
```

主なオプション:

| オプション | 説明 |
|---|---|
| `--asof 2026-03` | 円グラフの基準年度末（省略時=直近の完了年度） |
| `--no-llm` | LLM調査をせずキャッシュのみ使用（APIキー不要） |
| `--refresh-research` | キャッシュを無視して再調査 |
| `--list-unresolved` | ルールで判定できない施設の一覧だけ表示 |
| `--output パス` | 出力ファイル名の指定 |

## グレード判定のしくみ（優先順）

1. **オーバーライド表** … 施設名の完全一致（表記ゆれは正規化して吸収）。
   グレード欄に **`除外`** と書くと、その施設を集計から外せます（行は残り
   部屋数0・判定根拠に元の室数を記録）。対象市町村区外の施設や、
   名称違いの重複掲載（再開発の新旧計画など）の除外に使います。
2. **カテゴリーフィルタ** … ビジネス/デラックス/シティ/リゾートホテル・旅館
   以外（ホステル・ゲストハウス・民宿等）は「不明・その他」
3. **ブランド一次分類** … `grade_rules.csv` のブランド名でマッチ
4. **小規模ルール** … 20室未満はD（既存リストのみ）
5. **LLM調査（Step1.5）** … Claude API + Web検索で「ボリュームゾーン
   （最も室数が多い主力客室タイプ）の面積」を調査し、面積要件で判定
   **D: 20㎡未満 / C: 20㎡以上 / B: 30㎡以上 / A: 40㎡以上**
6. それでも不明 → 「不明・その他」+ 要確認フラグ（黄色行）

### 再現性（毎回同じ回答になる仕組み）

LLMの調査結果は `grade_research_cache.csv` に施設単位で永続保存され、
2回目以降は**必ずキャッシュが再利用**されます（API再呼び出しは
`--refresh-research` 指定時のみ）。キャッシュには面積・根拠・出典が
記録されるため、監査ログと手修正の場を兼ねます。判定が違うと思ったら
このCSVの「判定グレード」を直せば以後その値が使われます。

## 出力Excelの構成（出力はこの1ファイルのみ）

| シート | 内容 |
|---|---|
| `{市名} 供給数推移` | 集計表 + 積み上げ棒グラフ + 円グラフ |
| 既存リスト | 施設別の部屋数・開業/閉業年度・グレード・判定根拠・要確認 |
| 新規供給リスト | 開業予定施設の同上 |
| 閉業リスト | 閉業施設・グレード・既存リストへの突合結果（参考） |
| 参考_ブランドグレード表 | grade_rules.csv の写し |
| 参考_施設別グレード表 | オーバーライドCSVの写し |
| 参考_LLM調査キャッシュ | grade_research_cache.csv の写し（面積・根拠・出典） |
| グレード判定フロー | グレード決定の手順を自然言語で図示した説明タブ |

- 集計表はリストシートへの **SUMIFS数式** です。生成後にリストシートで
  グレード・開業年度・閉業年度を手修正すると、表とグラフが自動で再計算されます。
- 既存/新規/閉業の各リストの**グレード列はプルダウン**（A/B/C/D/不明・その他）で
  選択・変更できます（閉業リストのグレードは参考表示で、集計には既存リスト側を使用）。
- 参考タブは入力データファイルの写しです。恒久的に判定を変える場合は
  元のCSVを編集して再実行してください。
- 重複掲載への対応: 既存リストで**同名かつ同室数**の行、新規供給リストで
  **同名**の行は、2件目を自動で集計除外（部屋数0・判定根拠に記録）します。
  室数が異なる同名行は「同名重複疑い」と表示されるので、内容を確認のうえ
  片方の部屋数を0にする等の対応をしてください。名称が異なる重複
  （再開発計画の新旧掲載など）はオーバーライド表の`除外`で対応できます。

## 集計ロジック

各年度Yの供給室数（年度末時点） =
「開業年度 ≤ Y かつ (閉業していない or 閉業年度 > Y)」の既存施設の部屋数合計
＋「開業年度 ≤ Y」の新規供給室数の累積。
開業日が不明な既存施設は2019年度以前の開業とみなします。

## 注意

- 入力CSV・グレード表・出力Excelは社内情報を含むため、このリポジトリに
  コミットしないこと（`.gitignore`で保護済み）。
- 分析はメトロエンジンのデータに完全準拠します（衛生行政報告例との突合は行いません）。
