# ホテル供給状況レポート作成マニュアル（VS Code 版）

メトロエンジンからダウンロードした3つのCSV（既存施設 / 新規開業予定施設 / 閉業施設）から、
**グレード別×年度別の供給室数推移（積み上げ棒グラフ）** と
**直近年度末のグレード別供給室数割合（円グラフ）** を含むExcelを、コマンド1発で生成します。

このマニュアルは、Pythonやターミナルに不慣れな担当者でも VS Code 上で最後まで完結できるよう、
**インストールから実行・確認・トラブル対処・他都市への展開まで**を詳細に記載しています。

> 対応OS: Windows / macOS どちらでも同じ手順で動きます（コマンドの細部だけ後述の通り読み替え）。

## 1. 全体像

```
メトロエンジンから3つのCSVをDL
        │
        ▼
  make_supply_report.py を実行
        │  ①グレード付け（表→カテゴリ→ブランド→小規模→AI面積調査）
        │  ②閉館の反映（閉業年度以降を供給から除外）
        │  ③年度別に積み上げ集計（SUMIFS）
        │  ④グラフ化（積み上げ棒＋円）
        ▼
  YYYYMMDD_供給レポート_〇〇市.xlsx（1ファイルに全部入り）
```

「秘伝のタレ」化していた作業を、**総合職の担当者が自分で回せる**ように1本のスクリプトにまとめたものです。
Excel関数（SUMIFS）で組んであるので、生成後にリスト上でグレードや年度を直せば、
表もグラフも**自動で再計算**されます。

## 2. 事前準備

### 2-1. VS Code をインストール
公式サイト <https://code.visualstudio.com/> からダウンロードしてインストールします。

### 2-2. Python をインストール（3.10 以上）
- **Windows**: <https://www.python.org/downloads/windows/> から入手。
  インストーラ最初の画面で **「Add python.exe to PATH」に必ずチェック**を入れる。
- **macOS**: <https://www.python.org/downloads/macos/> から入手（または `brew install python`）。

インストール後、VS Code のメニュー **ターミナル → 新しいターミナル** を開き、次で確認：

```bash
python --version    # Windows（3.10 以上と表示されればOK）
python3 --version   # macOS
```

> 以降、コマンド例は `python` で書きます。macOS では `python` を `python3`、`pip` を `pip3` に読み替えてください。

### 2-3. VS Code の拡張機能
- **Python**（Microsoft製）… 必須級。VS Code 左の拡張機能アイコンから「Python」を検索して install。
- （任意）**Excel Viewer** … VS Code 内で xlsx をプレビューしたいとき。

## 3. プロジェクトのフォルダ構成

任意の作業フォルダ（例：`ホテル供給`）を作り、VS Code で **ファイル → フォルダを開く** で開きます。
最終的に次のように配置します（★＝あなたが用意/DLするファイル）：

```
ホテル供給/
├── make_supply_report.py          … 本体スクリプト（リポジトリからコピー）
├── grade_rules.csv                … ブランド→グレード表（受け渡し）
├── 那覇市_grade_overrides.csv      … 施設別グレード表（受け渡し・任意）
├── grade_research_cache.csv       … AI調査キャッシュ（受け渡し・自動更新）
├── ★既存.csv                       … メトロエンジンDL（既存施設）
├── ★新規.csv                       … メトロエンジンDL（新規開業予定施設）
└── ★閉業.csv                       … メトロエンジンDL（閉業施設）
```

> `make_supply_report.py` と3つのデータCSV（grade_rules / *_overrides / research_cache）は、
> 開発元から受け取ったものを**同じフォルダ**に置いてください。これらは社内情報を含むため、
> GitHub等の公開リポジトリには入れないでください。

## 4. 必要ファイルの配置

1. `make_supply_report.py` を作業フォルダにコピー。

3. メトロエンジンから落とした3つのCSVを同じフォルダに置き、分かりやすい名前
   （例：`既存.csv` `新規.csv` `閉業.csv`）にしておく。

## 6. 入力CSVの形式

列名は**部分一致**で自動認識するので、多少の表記違いは吸収されます。想定する主な列：

| ファイル | 認識する主な列（部分一致） |
|---|---|
| 既存施設 | `施設名` / `住所` / `カテゴリー` / `部屋数` / `開業` / `プライス・インデックス` |
| 新規開業予定 | `ホテルID` / `施設名` / `住所` / `部屋数`（(推定)部屋数でも可） / `竣工・開業予定日` |
| 閉業施設 | `閉業施設名`（施設名でも可） / `住所` / `部屋数` / `閉業日` / `カテゴリー` |

補足：
- 文字コードは UTF-8（BOM付きでも可）を想定。メトロエンジンのDLはそのまま使えます。
- 閉業CSVの部屋数が「集計中」の場合、既存リストに同名があればその部屋数で自動補完します。
- 日付は `YYYY-MM-DD` / `YYYY/M/D` / `YYYY年` いずれも解釈します。年しか無い場合は年央扱い。

## 7. 実行方法

### 7-1. 基本形（AI調査あり・APIキー設定済みの場合）

```bash
python make_supply_report.py --city 那覇市 \
  --existing 既存.csv --new 新規.csv --closed 閉業.csv \
  --grade-overrides 那覇市_grade_overrides.csv
```

### 7-2. AIを使わない（キャッシュのみ・APIキー不要）

初回に開発元が調査済みのキャッシュ（`grade_research_cache.csv`）を使うだけなら `--no-llm`：

```bash
python make_supply_report.py --city 那覇市 \
  --existing 既存.csv --new 新規.csv --closed 閉業.csv \
  --grade-overrides 那覇市_grade_overrides.csv --no-llm
```

### 7-3. 円グラフの基準年度末を指定

省略時は「直近の完了年度末」。明示するなら `--asof`（例：2025年度末＝2026年3月）：

```bash
... --asof 2026-03
```

実行が終わると、フォルダに **`YYYYMMDD_供給レポート_那覇市.xlsx`**（冒頭は実行日の西暦8桁）が
できます。VS Code のエクスプローラでダブルクリック → Excel で開けます。

### 7-4. 実行前に「AIで調べる対象」だけ確認したいとき

```bash
... --list-unresolved --no-llm
```

ルール（表・カテゴリ・ブランド・小規模）で決まらず**AI調査に回る施設だけ**が一覧表示されます。

## 8. 出力Excelの見方

1ファイルに次のタブが入ります：

| タブ | 内容 |
|---|---|
| **〇〇市 供給数推移** | 集計表＋積み上げ棒グラフ＋円グラフ（これがメインの成果物） |
| **既存リスト** | 施設別の部屋数・開業/閉業年度・グレード・判定根拠・要確認。**グレード列はプルダウン** |
| **新規供給リスト** | 開業予定施設の同上。**グレード列はプルダウン** |
| **閉業リスト** | 閉業施設・グレード・既存リストへの突合結果。**グレード列はプルダウン**（参考） |
| **参考_ブランドグレード表** | `grade_rules.csv` の写し |
| **参考_施設別グレード表** | オーバーライドCSVの写し |
| **参考_LLM調査キャッシュ** | AI調査結果（面積・根拠・出典） |
| **グレード判定フロー** | グレード決定手順を自然言語で図示した説明 |

ポイント：
- 集計表のセルは、リストシートへの **SUMIFS 数式**です。**リストのグレードや年度を書き換えると、
  表とグラフが即座に再計算**されます（値貼り付けではありません）。
- **要確認（黄色行）** … AIでも面積が確認できなかった施設。人の目で確認してグレードを直してください。
- **「同名重複疑い」** … メトロエンジン側で同名が重複掲載されている可能性。二重計上になるため、
  片方の部屋数を0にする等の対応をしてください。

## 9. グレード判定のしくみ（詳細）

1施設ずつ、上から順に条件をあてはめ、**あてはまった時点で確定**（以降のステップに進みません）。
これは出力Excelの「グレード判定フロー」タブと同じ内容です。

| STEP | 条件 | 結果 |
|---|---|---|
| 1 | 参考_施設別グレード表に**同じ施設名**があるか（過去資料の個別指定） | あれば その値で確定 |
| 2 | カテゴリーが宿泊主体型（ビジネス/デラックス/シティ/リゾートホテル・旅館）か<br>※新規供給はカテゴリー欄が無いのでスキップ | 対象外（ホステル/ゲストハウス/民宿/貸別荘/カプセル等）は**不明・その他**で確定 |
| 3 | 参考_ブランドグレード表の**ブランド名**に一致するか（チェーン・外資） | 一致すれば そのグレードで確定 |
| 4 | 客室数 **20室未満**か（明らかに小規模。既存リストのみ） | はい なら **D** で確定 |
| 5 | 上記で決まらない独立系 → **AI＋Web検索で主力客室（ボリュームゾーン）の面積を調査** | 面積要件：**A=40㎡以上 / B=30㎡以上 / C=20㎡以上 / D=20㎡未満**。面積優先、単価は補助。確信度が低ければ「要確認」 |
| 6 | STEP5でも不明 | **不明・その他**＋**要確認**（黄色） |

### 面積の考え方（重要）
- 「最低面積」ではなく **ボリュームゾーン（最も室数が多い主力タイプ）の面積**で判断します。
  スイート等に付随する狭小部屋を基準にすると誤判定になります。
- じゃらん/楽天/一休/公式HP で客室タイプ別の面積・室数を確認します。
- どうしても不明なときだけ、販売単価（ADRや素泊り最安）を補助的に参考にします。
  単価はダイナミックプライシングで変動が大きいため、優先度は面積より下です。

### 閉館（閉業）の反映
- 閉業日から**閉業年度**を求め、既存リストに突合して既存側へ付与します。
- 集計では **閉業年度以降はその施設を供給から除外**します（過大計上を防ぐ）。
- 既存リストに現れない閉業施設は、そもそも供給に計上されていないため集計対象外です。

### 年度別集計（4月〜翌3月）
- 各年度Yの供給室数 ＝
  「開業年度 ≤ Y かつ（未閉業 または 閉業年度 > Y）」の既存部屋数合計
  ＋「開業年度 ≤ Y」の新規供給室数の累積。
- 開業日が空欄の既存施設は「2019年度以前開業」とみなします。

## 10. LLM調査（AI＋Web検索）を有効にする

STEP5 のAI調査を**自分の環境で動かす**には、Anthropic の API キーが必要です。
（開発元が調査済みの都市を使うだけなら `--no-llm` でOK。キー不要。）

### 10-1. APIキーの入手
Anthropic のコンソール <https://console.anthropic.com/> でAPIキーを発行します。

### 10-2. 環境変数に設定

**Windows（PowerShell）**：
```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-xxxxx"     # そのターミナルのみ有効
# 恒久設定：
setx ANTHROPIC_API_KEY "sk-ant-xxxxx"        # 新しいターミナルから有効
```

**macOS / Linux（bash/zsh）**：
```bash
export ANTHROPIC_API_KEY="sk-ant-xxxxx"      # そのターミナルのみ
# 恒久設定は ~/.zshrc 等に上記 export 行を追記
```

### 10-3. 実行
`--no-llm` を付けずに実行すると、ルールで決まらなかった施設だけを1件ずつWeb検索で調べ、
結果を `grade_research_cache.csv` に保存します。**次回以降はキャッシュを使い、同じ回答**になります。

```bash
python make_supply_report.py --city 那覇市 \
  --existing 既存.csv --new 新規.csv --closed 閉業.csv \
  --grade-overrides 那覇市_grade_overrides.csv
```

- 再調査したいときだけ `--refresh-research` を付けます。
- APIキーが未設定なら、このステップは自動スキップされ、該当施設は「要確認」になります（動作は継続）。
- モデルを変えたいときは `--llm-model claude-opus-4-8` のように指定（既定でOK）。

## 11. 判定を手で直す

やり方は3通り。用途で使い分けます。

1. **その場だけ直す（一番かんたん）**
   出力Excelの 既存/新規/閉業 リストの**グレード列プルダウン**から選び直す。
   → 集計表・グラフが即再計算。ただし再実行すると元に戻ります。

2. **恒久的に直す（次回以降も反映）**
   - 施設単位 … `〇〇市_grade_overrides.csv` に `施設名, グレード` の行を追加/修正。
   - ブランド単位 … `grade_rules.csv` に `ブランド, グレード, 適用範囲, 備考` を追加。
   - AI調査結果 … `grade_research_cache.csv` の「判定グレード」列を直す。
   その後スクリプトを再実行。

3. **面積の閾値やカテゴリー定義を変える**
   `make_supply_report.py` 冒頭付近の定数を編集（上級者向け）：
   - `AREA_THRESHOLDS`（A/B/Cの面積境界）
   - `TARGET_CATEGORIES`（宿泊主体型とみなすカテゴリー）

> 例：面積要件だと STORYLINE 瀬長島 と EXES 那覇 は「A」ですが、単価感で「B」にしたい場合は、
> `grade_research_cache.csv` の該当行の判定グレードを B に書き換えて再実行します。

## 12. 出力結果の検証

資料にそのまま添付する前に、最低限これを確認します（VS Code のターミナルで実行できる簡易チェックも用意）。

- **開けるか**：xlsx をダブルクリックし、グラフ・表・各タブが表示されるか。
- **合計が妥当か**：「供給数推移」タブの合計行が、想定規模（例：那覇市で2万室台）か。
- **閉業が効いているか**：閉業年度の翌年度で、その施設の室数分だけ合計が減っているか。
- **要確認/重複**：黄色行と「同名重複疑い」を確認し、必要なら手修正。

ターミナルでの簡易検証例（Python）：
```bash
python - <<'PY'
import openpyxl, zipfile
f = "20260703_供給レポート_那覇市.xlsx"       # 実際のファイル名に変更
print("zip整合性(NoneならOK):", zipfile.ZipFile(f).testzip())
wb = openpyxl.load_workbook(f)
print("シート:", wb.sheetnames)
ws = wb["那覇市 供給数推移"]
print("合計行:", [ws.cell(row=10, column=c).value for c in range(1, 12)])
PY
```

## 13. 別の都市で作る

対応の中心は**受け渡しデータの都市差し替え**です。

1. メトロエンジンで対象都市の3CSV（既存/新規/閉業）をDL。
2. その都市に施設別グレード表があれば `〇〇市_grade_overrides.csv` として用意（無くても可）。
3. `grade_rules.csv`（ブランド表）は全国共通なのでそのまま使えます。
4. 実行：
   ```bash
   python make_supply_report.py --city 大阪市 \
     --existing 既存.csv --new 新規.csv --closed 閉業.csv \
     [--grade-overrides 大阪市_grade_overrides.csv]
   ```
5. ルールで決まらない施設はAI調査され、結果は同じ `grade_research_cache.csv` に
   都市名付きキーで蓄積されます（都市が違えば別エントリ）。

> エリア（首里/真和志/おもろまち/小禄/その他 等）の付与が必要な都市もありますが、
> 本ツールの集計はグレード×年度が対象です。エリア別内訳が要る場合は開発元に相談してください。

## 14. トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| `python: command not found` | Python 未インストール、またはPATH未設定。§2 を再確認。macOSは `python3`。 |
| `ModuleNotFoundError: openpyxl` | `python -m pip install openpyxl` を実行。 |
| `ModuleNotFoundError: anthropic` | AI調査を使うときのみ必要。使わないなら `--no-llm` を付ける。 |
| 文字化けする | 入力CSVがUTF-8か確認。ExcelでCSV保存すると別コードになる場合あり。メトロエンジンのDLをそのまま使う。 |
| グレードが全部「不明・その他」 | grade_rules.csv / overrides / research_cache が同じフォルダに無い可能性。§4 を確認。 |
| グラフの文字が重なる/見えない | 最新版のスクリプトを使う（薄い区分のラベルは非表示、大区分のみ内側表示に調整済み）。 |
| API 認証エラー（401等） | `ANTHROPIC_API_KEY` が未設定/誤り。§10。使わないなら `--no-llm`。 |
| 合計が過大に見える | 閉業CSVが入っていない可能性。3つ目の `--closed` を指定しているか確認。 |
| ファイル名の日付を変えたい | `--output 名前.xlsx` で任意名にできる。 |

## 15. コマンドライン オプション一覧

| オプション | 説明 |
|---|---|
| `--city` | （必須）都市名。ファイル名・タイトル・キャッシュのキーに使う |
| `--existing` | （必須）既存施設CSV |
| `--new` | （必須）新規開業予定施設CSV |
| `--closed` | （必須）閉業施設CSV |
| `--grade-overrides` | 施設名→グレードのオーバーライドCSV（任意） |
| `--grade-rules` | ブランド→グレード表CSV（既定：同フォルダの grade_rules.csv） |
| `--research-cache` | AI調査キャッシュCSV（既定：同フォルダの grade_research_cache.csv） |

## 16. 用語と背景

- **グレード**：A=ラグジュアリー / B=ハイクラス（アップ・アッパースケール）/ C=アッパーミドル /
  D=バジェット（ミッドスケール・エコノミー）/ 不明・その他。社内のグレードリストに準拠。
- **ボリュームゾーン**：最も客室数が多い主力の客室タイプ。グレード判定の面積はここで測る。
- **年度**：4月〜翌3月。開業年度・閉業年度ともこの区切り。
- **メトロエンジン準拠**：分析はメトロエンジンのデータに完全準拠し、
  厚労省「衛生行政報告例」との突合（看做簡易宿所の調整）は本ツールでは行いません。
- **資格情報の取り扱い**：メトロエンジンのログインID/パスワード等は、生成物やリポジトリに
  絶対に含めないでください。
